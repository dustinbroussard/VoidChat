# Void Chat

Void Chat is a React 19 chat client with a minimal AMOLED-style UI, persona-aware conversations, installable PWA support, and an OpenRouter-backed API layer.

## What It Does

- Runs a local chat UI with conversation history saved in `localStorage`
- Lets users switch between OpenRouter models and optionally filter to free models
- Supports personas with custom system prompts and saved persona-specific memory summaries
- Provides light and dark themes
- Ships with a service worker and web app manifest for installable PWA behavior
- Includes both:
  - a local Express + Vite dev server in [`server.ts`](/home/dustin/Projects/VoidChat/server.ts)
  - Vercel serverless API handlers in [`api/`](/home/dustin/Projects/VoidChat/api)

## Stack

- React 19
- TypeScript
- Vite
- Express
- Tailwind CSS v4
- OpenRouter API
- Vercel serverless functions for production API routes

## Environment Variables

The current app uses OpenRouter, not Gemini.

Create `.env.local` or `.env` with:

```env
OPENROUTER_API_KEY=your_openrouter_api_key
APP_URL=http://localhost:3000
OPENROUTER_APP_TITLE=Void Chat
```

Notes:

- `OPENROUTER_API_KEY` is required for both local development and production API calls.
- `APP_URL` is used as the OpenRouter `HTTP-Referer`. For local dev, `http://localhost:3000` matches the Express server.
- `OPENROUTER_APP_TITLE` is optional. If omitted, the app defaults to `Void Chat`.

## Local Development

Prerequisites:

- Node.js 20+
- npm

Install dependencies:

```bash
npm install
```

Start the local app:

```bash
npm run dev
```

This runs [`server.ts`](/home/dustin/Projects/VoidChat/server.ts), which starts an Express server on `http://localhost:3000`, mounts the API routes, and serves the Vite app through middleware.

## Available Scripts

```bash
npm run dev
npm run build
npm run preview
npm run lint
```

- `npm run dev`: starts the Express + Vite development server
- `npm run build`: builds the frontend into `dist/`
- `npm run preview`: previews the static Vite build
- `npm run lint`: runs `tsc --noEmit`

`npm run preview` only serves the built frontend. If you need working `/api/*` routes locally, use `npm run dev`.

## Production Deployment

The repo is set up for Vercel:

- static frontend built from Vite
- `/api/models` and `/api/chat` handled by Vercel functions
- SPA rewrites configured in [`vercel.json`](/home/dustin/Projects/VoidChat/vercel.json)

Set these environment variables in Vercel:

- `OPENROUTER_API_KEY`
- `APP_URL`
- `OPENROUTER_APP_TITLE` optional

For `APP_URL`, use the deployed site origin, for example `https://your-app.vercel.app`.

## Current Behavior

- Conversations are stored in browser `localStorage`
- Personas are stored in browser `localStorage`
- Theme and selected model are stored in browser `localStorage`
- The app fetches available models from OpenRouter at runtime
- Chat requests are proxied through local or serverless API routes so the browser never needs the OpenRouter key directly
- Persona memory summaries are generated by the selected model and saved back onto the active persona
- The service worker caches the app shell and static assets, but API responses are not cached for offline chat
